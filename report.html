Ford GoBike Trip Data Analysis Report

1. High-Level Overview

In this analysis, we explore the Ford GoBike trip data from February 2019 to uncover usage patterns, understand user demographics, and improve bike-sharing services. Our objectives are to identify trends in trip durations, popular stations, and user behavior, and to derive actionable insights to enhance service efficiency.

2. Description of Input Data

The dataset 201902-fordgobike-tripdata.csv contains information about bike trips from February 2019. Key features include:

  • duration_sec: Duration of the trip in seconds
  • start_time: Start time of the trip
  • end_time: End time of the trip
  • start_station_id: ID of the start station
  • end_station_id: ID of the end station
  • bike_id: ID of the bike used
  • user_type: Type of user (e.g., Subscriber, Customer)
  • member_birth_year: Birth year of the member
  • member_gender: Gender of the member

3. Strategy for Solving the Problem

  • Data Cleaning: Address missing values and outliers.
  • Exploratory Data Analysis (EDA): Examine distributions of trip durations, frequencies of start/end stations, and user demographics.
  • Data Preprocessing: Transform time columns, normalize numerical features.
  • Modeling: Apply clustering algorithms to identify usage patterns.
  • Visualization: Create charts and graphs to visualize findings.

4. Discussion of the Expected Solution

We anticipate discovering patterns in bike trip durations, peak usage times, and popular stations. We expect to find differences in trip behaviors based on user types and genders, which could inform targeted improvements for the bike-sharing service.

5. Metrics with Justification

  • Silhouette Score: Measures how similar an object is to its own cluster compared to other clusters, used to evaluate clustering quality.
  • Trip Duration Statistics: Provides insights into average trip durations and usage trends.

6. Exploratory Data Analysis (EDA)

During EDA, the following observations were made:

  • Trip Duration: The majority of trips are under 20 minutes, with a few outliers having significantly longer durations.
  • Station Popularity: Some stations are much more frequently used as start or end points, indicating high traffic areas.
  • User Demographics: Most users are male and fall within the 25-40 age range.

7. Data Preprocessing

  • Handling Missing Values: Filled missing values in categorical columns with the most frequent value and in numerical columns with the mean.
  • Time Transformation: Converted start_time and end_time to datetime format for analysis.
  • Normalization: Scaled numerical features to a standard range using StandardScaler.

8. Modeling

We used KMeans clustering to identify patterns in bike usage. MiniBatchKMeans was selected due to its efficiency with large datasets. The optimal number of clusters was determined using the Elbow Method, which suggested 3 clusters for the data.

9. Hyperparameter Tuning

  • Number of Clusters: Tested values from 2 to 10, with 3 clusters providing the best results based on the Elbow Method.

10. Results

The KMeans clustering revealed three distinct user segments:

  • Frequent Users: High number of short trips, frequent usage of central stations.
  • Moderate Users: Moderate trip frequency, with balanced usage across stations.
  • Occasional Users: Few trips, generally longer in duration, less frequent use of central stations.

Visualizations include:

  • Trip Duration Distribution: Histogram showing the frequency of different trip durations.
  • Station Popularity: Bar chart illustrating the number of trips starting and ending at each station.
  • Clustering Results: Scatter plot of clusters with trip durations.

12. Conclusion

The analysis identifies distinct usage patterns among bike-sharing users. Frequent users have short, regular trips, while occasional users take longer trips less frequently. The insights can guide targeted promotions and station improvements.

13. Improvements

Future analysis could incorporate additional factors such as weather conditions or seasonal variations. Including more recent data may also provide insights into current usage trends.
